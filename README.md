#  Крестики-нолики (Tic-Tac-Toe)

Классическая логическая игра для двух игроков, реализованная на чистом JavaScript с манипуляциями DOM.

## 1. Описание игры
- **Правила игры**: Игроки поочередно ставят свои символы (`X` и `0`) на пустые клетки поля 3×3. Цель — первым выстроить три своих символа в ряд по горизонтали, вертикали или диагонали.
- **Логика победы/поражения**: Игра завершается сразу при обнаружении выигрышной комбинации у текущего игрока. Если все клетки заполнены, но комбинация не собрана, фиксируется ничья. После завершения игры поле блокируется до перезапуска.
- **Особенности реализации**: Проект не использует сторонние библиотеки или фреймворки. Состояние хранится в памяти, а интерфейс синхронизируется напрямую через `textContent`. Обработка событий делегируется через перебор NodeList, что обеспечивает чистую и легковесную архитектуру.

## 2. Игровая механика
- **Как определяется победитель?** После каждого хода вызывается функция `checkResult()`, которая сверяет текущее состояние поля с массивом выигрышных линий. При совпадении игра останавливается, и в статус-блок выводится сообщение о победе.
- **Как обрабатываются клики?** На каждую ячейку (`.cell`) навешан слушатель `click`. При клике сначала проверяются два условия: занята ли клетка и активна ли игра. Если условия выполнены, символ записывается в массив состояния и в DOM, после чего запускается проверка результата.
- **Как работает перезапуск?** Кнопка `restart` вызывает функцию `restartGame()`, которая сбрасывает все переменные в исходные значения, очищает текстовое содержимое ячеек и обновляет статус. Страница не перезагружается, состояние инициализируется заново в памяти.

## 3. Структура данных
- **Массив `board`**: Одномерный массив из 9 строк, где индексы `0–8` соответствуют ячейкам поля (нумерация слева-направо, сверху-вниз). Пустая строка `""` означает свободную клетку, `"X"` или `"0"` — занятую.
- **Условия победы**: Массив `winConditions` содержит 8 подмассивов. Каждый подмассив хранит три индекса, образующие выигрышную линию (3 горизонтали, 3 вертикали, 2 диагонали).
- **Как хранится состояние игры?** Состояние распределено по трем глобальным переменным:
  - `currentPlayer` (строка) — очередь хода (`"X"` или `"0"`).
  - `board` (массив) — текущая расстановка символов на поле.
  - `gameActive` (boolean) — флаг активности игры. При `false` блокирует обработку новых кликов.

## 4. Алгоритмы
- **Алгоритм проверки победы**: Цикл `for` последовательно перебирает все комбинации из `winConditions`. С помощью деструктуризации извлекаются индексы `[a, b, c]`. Проверяется условие: `board[a] && board[a] === board[b] && board[a] === board[c]`. Если оно истинно, ставится флаг `roundWon = true`, цикл прерывается (`break`), и игра завершается.
- **Алгоритм определения ничьей**: Используется метод `Array.prototype.includes("")`. Если массив `board` не содержит пустых строк (`!board.includes("")`), а победитель ещё не найден, фиксируется ничья и игра останавливается.
- **Алгоритм смены игрока**: Реализован через тернарный оператор:  
  `currentPlayer = currentPlayer === "X" ? "0" : "X";`  
  Если ходил игрок `X`, очередь передаётся игроку `0`, и наоборот. После смены статус-текст обновляется автоматически.
